# Task - AI Workspace Startup Validation

## Objective

Validate the new `.AI` startup architecture by performing a project review using the startup process defined in `Start_Here.md`.

The goal of this task is to verify that the startup workflow, role selection, procedures, standards, and execution-record process operate as intended.

## Scope

Review the `.AI` workspace and the startup framework only.

Include:

- Startup workflow
- Roles
- Procedures
- Standards
- Prompt loading
- Execution-record behavior
- Overall organization

Do not review unrelated project source code.

## Required Startup

Follow the startup process defined by `Start_Here.md`.

Do not bypass the startup workflow.

## Required Outputs

Create the following documents:

- `Results/AI-Startup-Review.md`
- `Results/AI-Startup-TODO.md`
- Execution record appropriate to the selected execution-record mode.

## Review Requirements

Evaluate:

- Correctness of the startup sequence.
- Circular references.
- Redundant or orphaned documents.
- Clarity of Roles, Procedures, and Standards.
- Opportunities to reduce unnecessary document loading.
- Opportunities to improve deterministic behavior.
- Missing components.
- Conflicting guidance.
- Startup performance.

Support findings with evidence where possible.

Distinguish observed facts from recommendations.

Rank recommendations as:

- Critical
- High
- Medium
- Low

## Execution Record

Use **Verbose** execution-record mode.

Record:

- Startup documents loaded.
- Selected role.
- Procedures loaded.
- Standards loaded.
- Commands executed.
- Files inspected.
- Artifacts created.
- Validation performed.
- Resource usage where available.
- Clearly distinguish observed events from reconstructed events.

## Completion Criteria

The task is complete when:

- The startup framework has been reviewed.
- The review report has been created.
- The TODO list has been created.
- A verbose execution record has been produced.
