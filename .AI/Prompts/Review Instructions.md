# Review Instructions

## Purpose

These instructions apply to every engineering review.

## Reading Requirements

Before writing **any*- report:

-   Read **every*- source file completely from beginning to end.
-   Do **not*- stop after the first few hundred lines.
-   Do **not*- summarize a file until you reach its final line.
-   If a file exceeds your normal chunk size, continue reading until the
    last line.
-   Several large files contain critical implementation details near the
    end.

## Analysis Requirements

-   First pass is information gathering only.
-   Do not form conclusions while reading.
-   Maintain a running list of observations.
-   Convert observations into recommendations only after the entire
    project has been examined.

## Review Philosophy

Treat the review as an engineering design review, **not*- a style
review.

Do not recommend cosmetic changes.

Recommend only improvements that materially improve:

-   Correctness
-   Reliability
-   Maintainability
-   Performance
-   Extensibility
-   Architecture
-   User experience
-   Production readiness

Produce no code changes unless explicitly requested.
