# Procedure - Validation

## Purpose

Perform a structured validation of completed work to verify that the requested task has been completed, applicable standards have been satisfied, and outputs are suitable for delivery.

## Applicability

Use this procedure whenever work has been completed and requires verification before it is delivered or recorded.

This procedure is especially applicable to:

- Documentation updates.
- Code reviews.
- Debugging investigations.
- Test execution.
- Execution records.
- Code generation.
- Configuration changes.
- Architecture documents.

## Prerequisites

Before using this procedure:

- The primary task must be complete.
- The active role must have authority to perform validation.
- The expected outputs must be available.
- Applicable standards must be identified.

## Inputs

Typical inputs include:

- User instructions.
- Generated outputs.
- Source documents or project files.
- Diagnostics and logs.
- Execution evidence.
- Applicable standards.
- Relevant project context.

## Required Standards

Load only the standards required for the current task, for example:

- Output Document Formatting Rules.
- Global Markdown Documentation Standard.
- Project coding standards.
- Execution Record Standard, when applicable.

## Invoked Procedures

This procedure normally does not invoke additional procedures.

If additional validation is required, invoke only the minimum necessary procedure and load it only once per run.

## Steps

1. Identify the scope of validation.
2. Confirm the requested task has been completed.
3. Verify compliance with applicable standards.
4. Verify outputs for correctness, completeness, and consistency.
5. Confirm referenced files and artifacts are identified.
6. Record any warnings, errors, deviations, or unresolved issues.
7. Produce a validation summary suitable for the calling procedure.

## Expected Outputs

Typical outputs include:

- Validation summary.
- Pass/fail status.
- List of validation findings.
- Outstanding issues, when applicable.
- Recommended follow-up actions.

## Validation

Validation should confirm that:

- The requested work is complete.
- Applicable standards have been satisfied.
- Outputs are internally consistent.
- Findings are supported by evidence.
- Outstanding issues are clearly identified.

## Completion Criteria

This procedure is complete when:

- Validation has been performed.
- Results have been documented.
- Any unresolved issues have been reported.
- The calling procedure can determine whether the task is ready for delivery.

## Notes

This procedure defines **how** completed work is verified. It does not define **who** performs the work. The active role defines authority and responsibility, while standards define the rules this procedure must satisfy.
